## 📋 요약

본 저장소는 Antora를 사용하여 홍익대학교 ROTC 55기 동기회 회칙을 버전별로 관리하는 문서화 프로젝트입니다. 각 브랜치는 특정 날짜부터 적용되는 회칙 버전을 나타냅니다.

---

## 🔍 주요 개선 필요 사항

### 1. **브랜치 구조 및 버전 관리 (중요도: 높음)**

#### 문제점:
- `antora-source.yml:8`에 `2023.1.1`, `2024.1.1` 브랜치만 정의되어 있음
- **2025.1.1** 및 **2026.1.1** 브랜치가 누락됨
- 회칙 수정이 없었더라도 연도별 버전 관리 측면에서 브랜치 생성 필요

#### 현재 코드:
```yaml
# antora-source.yml:8
branches: 2024.1.1, 2023.1.1
```

#### 권장 수정사항:
```yaml
# antora-source.yml:8
branches: 2026.1.1, 2025.1.1, 2024.1.1, 2023.1.1
```

#### 이유:
- Antora는 브랜치별로 문서 버전을 생성하므로, 사용자가 각 연도의 회칙을 명확하게 확인할 수 있어야 함
- 회칙 변경이 없더라도 "2025.1.1부터 회칙 수정 없음" 메시지를 포함한 버전 페이지 제공으로 투명성 확보
- 향후 특정 연도 회칙을 찾을 때 혼란 방지

---

### 2. **버전 문서 일관성 (중요도: 높음)**

#### 문제점:
- `2024.1.1` 브랜치에는 `version.adoc` 파일이 존재
- `2023.1.1` 브랜치에는 `version.adoc` 파일이 없음
- 브랜치 간 문서 구조 불일치

#### 권장 수정사항:
모든 버전 브랜치에 `modules/ROOT/pages/version.adoc` 파일을 추가하여 일관성 유지:

**2025.1.1 브랜치용:**
```adoc
= 회칙 변경사항

== 2025. 01. 01

====
회칙 수정 없음

2024.01.01 회칙을 그대로 적용합니다.
====
```

**2026.1.1 브랜치용:**
```adoc
= 회칙 변경사항

== 2026. 01. 01

====
회칙 수정 없음

2024.01.01 회칙을 그대로 적용합니다.
====
```

### 3. **README.adoc 개선 (중요도: 중간)**

#### 문제점:
- `README.adoc:1`가 거의 비어있음 (단 한 줄: `= 133 ROTC 55`)
- 프로젝트 설명, 사용법, 구조 설명이 전혀 없음

#### 권장 수정사항:
```adoc
= 홍익대학교 ROTC 55기 동기회 회칙

== 소개

본 저장소는 홍익대학교 ROTC 55기 동기회 회칙을 관리하는 공식 문서 저장소입니다.
Antora를 사용하여 버전별 회칙을 체계적으로 관리합니다.

== 브랜치 구조

각 브랜치는 해당 날짜부터 적용되는 회칙 버전을 나타냅니다:

- `2023.1.1`: 2023년 1월 1일부터 적용
- `2024.1.1`: 2024년 1월 1일부터 적용
- `2025.1.1`: 2025년 1월 1일부터 적용 (수정 없음)
- `2026.1.1`: 2026년 1월 1일부터 적용 (수정 없음)

== 문서 확인

배포된 문서는 GitHub Pages를 통해 확인할 수 있습니다.

== 로컬 빌드

```bash
npm install antora
npx antora antora-source.yml
```

== 회계 정보

회계 처리 내역은 link:회계처리.md[회계처리.md]를 참고하세요.
```

### 4. **파일 위치 및 정리 (중요도: 중간)**

#### 문제점:
- `홍익대학교 ROTC 55기 동기회 회칙_rev2(2024.1.1부터적용).hwp` 파일이 루트에 위치
- 이 파일이 main 브랜치에만 있고, 문서 브랜치(2024.1.1 등)에도 중복 존재
- 원본 문서와 Antora 문서 간의 관계가 명확하지 않음

#### 권장 수정사항:
1. 원본 HWP 파일을 별도 디렉토리로 이동:
   ```
   /original-docs/
     └── 홍익대학교 ROTC 55기 동기회 회칙_rev2(2024.1.1부터적용).hwp
   ```

2. `.gitignore` 업데이트:
   ```
   # .gitignore
   build/
   node_modules/
   *.hwp  # HWP 파일은 original-docs/에서만 관리
   ```

3. `회계처리.md` 파일을 Antora 문서로 변환하거나 별도 관리 필요

### 5. **GitHub Actions 워크플로우 개선 (중요도: 낮음)**

#### 문제점:
- `.github/workflows/publish.yml:4`가 `main` 브랜치에서만 트리거됨
- 실제 문서는 버전별 브랜치(`2024.1.1` 등)에 존재
- main 브랜치 역할이 불명확함

#### 현재 코드:
```yaml
# .github/workflows/publish.yml:4
on:
  push:
    branches: [main]
```

#### 권장 수정사항:
현재 구조가 의도적이라면 문제없으나, 워크플로우 트리거 조건을 명확히 문서화 필요:

**옵션 1**: main 브랜치를 설정 전용 브랜치로 유지 (현재 구조)
- `antora-source.yml`에서 원격 브랜치를 참조하므로 main에서만 빌드하는 것이 맞음
- 이 경우 README에 명시 필요

**옵션 2**: 특정 브랜치 업데이트 시에도 빌드
```yaml
on:
  push:
    branches:
      - main
      - '202*.1.1'  # 2023.1.1, 2024.1.1 등
```

### 6. **antora-source.yml 설정 검토 (중요도: 낮음)**

#### 문제점:
- `antora-source.yml:7`에 주석 처리된 로컬 URL이 있음
- 배포 시 원격 저장소 사용, 로컬 개발 시 전환 필요성 불명확

#### 현재 코드:
```yaml
# antora-source.yml:6-7
- url: https://github.com/133rotc/rules
# - url : ./
```

#### 권장 수정사항:
주석을 명확하게 작성:
```yaml
content:
  sources:
  - url: https://github.com/133rotc/rules
    # 로컬 개발 시: url: ./ 로 변경하여 사용
    branches: 2026.1.1, 2025.1.1, 2024.1.1, 2023.1.1
```
